Code Review 不是挑刺，也不是风格争论。它应该帮助团队降低缺陷率，并让关键知识在团队内流动。

## 审什么

我通常优先看正确性、边界条件、可维护性、测试覆盖和是否符合既有架构。

## 怎么反馈

好的 Review 反馈应该具体、可执行，并且区分必须修改和建议优化。
